# E-book List Manager

Kindle、DMM Books、ebookjapan のライブラリページから書籍情報を自動収集・管理し、CSVに出力する Chrome 拡張です。データは外部へ送信せず、Chrome拡張のローカルストレージに保存します。

## 対応ページ

- `https://read.amazon.co.jp/kindle-library`
- `https://book.dmm.com/shelf/?tab=library&page=1`（ページ番号は任意）
- `https://book.dmm.com/product/<シリーズID>/<コンテンツID>/`
- `https://book.dmm.com/thanks/?order_id=<注文ID>`
- `https://ebookjapan.yahoo.co.jp/bookshelf/`
- `https://ebookjapan.yahoo.co.jp/bookshelf/stories/`

対象ページを開くと自動収集します。Kindleの遅延読み込み、DMM Booksとebookjapanの動的描画に追随するため、DOMの変更を継続監視します。データはシリーズ単位に集約され、同じシリーズの巻数と書籍IDは配列として保持されます。

## インストール

1. GitHub Releases から最新の `e-book-list-manager.zip` をダウンロードする
2. ZIPを任意の場所へ展開する
3. `chrome://extensions` を開く
4. 「デベロッパー モード」を有効化する
5. 「パッケージ化されていない拡張機能を読み込む」を押し、展開した `extension` フォルダを選ぶ
6. Kindle / DMM Books / ebookjapan にログインして対応ページを開く

ツールバーの拡張アイコンから冊数とシリーズ数の確認、再解析、一覧表示ができます。「一覧 / CSV」画面で検索、サービス絞り込み、UTF-8 BOM付きCSV出力、シリーズ単位・サービス単位・全件の削除ができます。削除時に除外リストへ追加すると、以降の収集でも同じシリーズを保存しません。

「取り込みガイド」では、対応URLの一覧とサービスごとの操作手順を確認できます。直接開けるURLにはリンクがあり、商品個別ページや購入完了ページは遷移方法を案内します。ガイド内容は各解析ルールの `importGuide` から生成されます。

## 解析ルール

オプション画面では次をJSONで編集できます。

- 対象URL（`urlPatterns`、`*` ワイルドカード対応）
- 書籍単位のCSSセレクタ（`bookSelector`）
- ラベル付き抽出項目の配列（`fields`）
- 正規表現によるタイトル補正（`titleCleanup`）
- 巻数抽出とシリーズ集約（`seriesGrouping`）
- ルールの有効・無効、新しいサイト用ルールの追加

任意JavaScriptは実行しません。ルール変更後は対象ページを再読み込みしてください。標準ルールはいつでも復元できます。

### ルールの並び

各サービスのルールは、上から次の順に並びます。

1. `id`: 保存データで使う変更不可の識別子
2. `name`: 画面に表示するサービス名
3. `enabled`: 有効・無効
4. `urlPatterns`: 対象URL
5. `importGuide`: 取り込みガイドに表示するURL・手順・補足
6. `parser`: `dom` または内蔵の `kindle`
7. `bookSelector`: 一冊分を囲む要素
8. `fields`: 抽出する項目
9. `titleCleanup`: 全タイトルに対する置換
10. `seriesGrouping`: シリーズIDまたはタイトルと巻数による集約

同じサービスの一覧ページと個別ページを別ルールにする場合は、個別ページ側に `collectionId` と `collectionName` を指定します。DMMでは `dmm-books-product` の結果を `collectionId: "dmm-books"` に保存するため、本棚の既存データと同じシリーズIDで統合されます。

### 抽出項目

```json
{
  "key": "title",
  "label": "タイトル",
  "selector": "img.book-cover",
  "value": "attribute",
  "attribute": "alt",
  "match": "\\s+第?[0-9]+巻$",
  "replace": ""
}
```

`value` は `text`（既定）、`attribute`、`html`。複数要素は `join` で連結します。`type: "array"` を指定すると、該当要素を文字列配列として保持します。DMMのステータスがこの形式です。

### Kindleの巻数とシリーズ集約

`seriesGrouping.volumePatterns` を上から評価し、最初に一致した正規表現の `volumeGroup` を巻数として取得します。`removeFromSeriesTitle: true` の場合、一致部分をタイトルから削除し、その結果が同じ書籍を同一シリーズとしてまとめます。

```json
{
  "description": "『第3巻』『３巻』",
  "match": "\\s*(?:第\\s*)?([0-9０-９]+)\\s*巻",
  "volumeGroup": 1,
  "removeFromSeriesTitle": true
}
```

特殊なタイトルは、この配列の先頭に専用パターンを追加できます。DMMはタイトルではなく `seriesId` を集約キーにします。

## 収集項目

サービス、書籍ID配列、シリーズID、タイトル、著者、カテゴリ、ステータス配列、所持巻数配列、詳細URL、表紙URL、収集元URL、最終確認日時です。

DMM Booksの一覧ページには所有巻数がないため、巻数は推測しません。「未購入続刊あり」「完結」など、カード上のラベルを `statuses` 配列として保存します。Kindleには対応するラベルがないため、ステータスは空配列です。

Kindleとebookjapanはタイトルから巻数を抽出して `ownedVolumes` 配列に保存します。一覧とCSVでは `[1,2,3,4,5,7,9,10,11,12,13,14,15]` を `1-5, 7, 9-15` と表示します。

Kindle、DMM Books、ebookjapan共通で、`分冊版`、`story03`、`【第4話】`、`2話` のような話数表記は同一シリーズにまとめ、ステータスに `分冊版` を追加します。`電子限定特典付き/つき`、`電子版限定特典付き`、`特典ペーパー付き` などの末尾特典表記はシリーズ名から外して集約します。

DMMの個別ページでは、シリーズ欄に表示された `購入済み` カードだけを解析します。カードのURLからシリーズIDとコンテンツID、タイトルから巻数を取得し、本棚ページで保存済みの同一シリーズへマージします。シリーズ欄がページ分割されている場合は、各ページを表示すると追加分が保存されます。未購入カードは所持巻数に含めません。

DMMの購入完了ページでは「今すぐ読む」と「予約済み作品」の商品を取得し、シリーズIDで既存データへマージします。配信開始予定の表示がある商品には `予約済み` ステータスを追加します。購入完了ページ自体に購入日時がないため、初めて拡張機能が取得した時刻を `firstSeenAt`（本棚追加日時）として保存します。

ebookjapanは同じ本棚URLにリストビューと本棚ビューがあります。URLではなくDOM内容で同じルールが両ビューを処理します。リストビューでは著者名と通常表紙を取得し、本棚ビュー（背表紙表示）では各巻の表紙altから所持巻数を補完します。著者名が取れているリストビューの書誌を優先するため、本棚ビューの空書誌や背表紙画像でタイトル・著者・表紙を上書きしません。リストビューの件数表示だけでは所持巻の範囲を確定できないため、巻数を正確に揃えるには両方のビューを表示してください。

ebookjapanの分冊・話数表記も同じ共通ルールで処理します。

ebookjapanの単話本棚（`/bookshelf/stories/`）では一覧の件数表示を所持話数として扱い、`1-N` の所持巻数として保存します。個別の欠番まではこの一覧からは判定しません。

## 一覧画面

一覧では、キーワードとサービスに加えて複数ステータスで絞り込めます。複数のステータスを選ぶと、選択したステータスをすべて持つシリーズを表示します。並び順は本棚追加順、最終確認順、タイトル、所持巻数、サービスに対応します。ステータスは個別のタグとして表示し、複数の書籍IDは先頭またはシリーズIDだけを表示して残りを `+件数` にまとめます。カーソルを合わせると全IDを確認できます。各行の操作メニューから手動ステータス編集とシリーズ単位の削除ができ、サービスを選択した状態ではそのサービスの全シリーズをまとめて削除できます。手動ステータスは収集で上書きされず、一覧・検索・CSVに反映されます。削除時に除外対象にしたシリーズは除外リストへ表示され、除外解除すると次回以降の収集対象に戻ります。

## 開発

```bash
npm install
npm test
npm run check
npm run package
```

テストは `tests/fixtures` の人工HTMLを読み、各サイトの抽出結果とCSVを検証します。実サービスから保存したHTMLには購入履歴やセッション情報が含まれる可能性があるため、リポジトリには追加しません。`npm run package` はJDKの `jar` コマンドを使い、`dist/e-book-list-manager.zip` を生成します。

## コントリビューション方針

このリポジトリでは外部からのPull Requestは受け付けていません。セキュリティ上、ブラウザ拡張の権限・DOM解析・取り込み対象URLに関する変更はメンテナ側で管理します。

欲しい機能や不具合報告はIssueで受け付けます。自分の用途に合わせて実装したい場合は、このリポジトリをForkして利用してください。

## 権限について

解析ルールで任意ドメインを追加できる要件のため、ホスト権限は `<all_urls>` です。コンテンツスクリプトはURLが有効ルールに一致しない限り、DOM監視もデータ送信も行いません。ネットワーク送信機能はありません。
